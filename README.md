## ğŸ”’ Responsible Disclosure Policy

I follow responsible disclosure practices:
1. **Identification** - Discover vulnerability through authorized testing
2. **Validation** - Confirm the vulnerability exists
3. **Reporting** - Submit detailed report via proper channels
4. **Coordination** - Allow reasonable time for remediation
5. **Disclosure** - Public disclosure only after fix is deployed

**Typical Timeline:**
- Day 0: Initial report submission
- Day 1-2: Triage/acknowledgment
- Day 7: Follow-up if no response
- Day 30: Request update
- Day 90: Public disclosure (if permitted)
